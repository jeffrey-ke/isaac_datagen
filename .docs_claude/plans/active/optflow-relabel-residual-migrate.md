# Relabel (near-duplicate merge) optflow `amazon` classes via residual migrate â€” no re-render

## Context

We want to **merge some near-duplicate `amazon` box classes** in the optical-flow
(`mode: optflow`) datasets (e.g. fold two visually-identical color families into one class).
The render dirs feed **both** the Stage-2/3 reference-seg pipeline **and** UFM optical-flow
training, so correctness of the class-keyed catalogs matters for both.

Key realization that shapes the whole approach: a **near-duplicate merge changes only the
class _string_** â€” not geometry, placement, or camera poses. Every rendered
`rgb`/`observation_depth`/`iid_mask`/`cam2world` is byte-identical before and after. The only
artifacts that depend on the label are:

- the per-frame `cid_mask` (a `uint8` class-id image), and
- the once-per-dir **class-keyed catalogs** in `OptFlowMetadata` (+ its nested
  `ObsMaskDescriptorMetadata`).

So instead of re-rendering 12 dirs through Isaac Sim (hours of GPU, not resumable, the
finalize/black-frame footguns), we do a **residual migrate** on disk â€” the repo's established
residual-write idiom (`relabel_classes.py`'s `only={"meta"}`, `migrate_descriptors_backbone
add-backbone`'s no-re-render refill). This preserves the already-baked `CleanDiftFinetunedDescriptor`
features (re-keyed, not recomputed) and runs in minutes on CPU. Downstream Stage-2 proposals,
Stage-3 inliers, and the train/val split are then regenerated by **existing** tools (none need Isaac).

## Correctness assumption (must hold)

For a merged class `N` formed from `{A, B}`, the migrate keeps **one** representative's per-class
catalog entry (reference RGB/depth/intrinsics/pose + DIFT descriptors) and warps it into **all**
merged instances via their preserved `class_to_l2w` rows. This is correct **only** because `A` and
`B` are near-duplicates (same shape/print) â€” exactly the merge kind chosen. It mirrors the renderer's
own "one canonical reference per class" rule (`reference_catalog`: first member by sorted name;
`OptFlowMetadata.finalize_metadata`: `members[0]`). **Do not use this path to merge structurally
different objects** â€” that would corrupt the UFM refâ†’obs warp.

## Scope â€” affected dirs

We merge `amazon` classes only. `amazon` instances are placed in **all 12 canonical optflow render
dirs**, and merging shrinks each dir's sorted class set (so even `kleenex`/`ycb` cids shift within a
dir). All 12 must be migrated:

- `mixed-persp/render000..005` (6)  Â· `shelf-optflow/render000,001,003` (3)  Â· `expanded-refseg/render000..002` (3)

The `*_viz_*` sibling dirs are derived viz, not training data â€” **skip them** (they are not in
`render_subdirs`). The migrate is a no-op for any dir whose present-class set is untouched by the
merge map (detect â†’ skip).

## The merge map (single source of truth)

Author one explicit YAML, `merge_amazon.yaml`, mapping each retired class to its survivor; classes
absent from the map are identity. Explicit + fail-loud (no guessing), per repo policy.

```yaml
# old_class: new_class   (omitted classes keep their name)
amazon_pistachio: amazon_green     # example near-dup folds â€” user fills in real pairs
amazon_tea:       amazon_teal
```

This same map drives both the asset relabel (step 0) and the dir migrate (step 1).

## Step 0 â€” Relabel the optflow asset metas (source of truth for future renders)

Apply the map to `assets/optflow_objects/amazon/meta/meta_*.yaml` (`meta["class"]`) so the assets and
the rendered data agree, and any future re-render reproduces the merged labels.

`relabel_classes.py` today is interactive **and hardcoded to `GraspableObject`**
(`GraspableObject.deserialize`, `isaac_datagen/src/isaac_datagen/relabel_classes.py:62`). Add a
small **non-interactive, map-driven** path that operates on the optflow assets:

- Deserialize via `OptFlowObject` (it has `meta` + `reference_image` and reuses
  `GraspableObject._serializers`, so `OptFlowObject.deserialize` round-trips; `objects.py:58`).
- For each sample whose `meta["class"]` is in the map, set the new class and
  `serialize(idx, dir, only={"meta"})` â€” residual, leaves usdz/png/npy untouched.
- Select the object type with an **explicit** flag (e.g. `--object-type optflow|graspable`), not a
  guess â€” matches the repo's explicit-selector policy.

(Optional: relabel the upstream `GraspableObject` source too, for symmetry â€” not load-bearing here.)

## Step 1 â€” Residual migrate of each render dir (`migrate_optflow_relabel.py`, NEW)

A new `isaac_datagen` script (`isaac-datagen-relabel-migrate <dataset_root> <merge_map.yaml>`) that,
per canonical render dir, rewrites `cid_mask` + the class-keyed catalogs in place. Reuse the
field-level seams the `add-backbone` migrate already uses (`SerializableSample.deserialize_field`,
`atomic_write_path`, `add_backbone_to_subfolder`, `fit_pca_basis` â€”
`migrate_descriptors_backbone.py`); **do not** monolithically re-serialize the multi-backbone
`SubfolderDict`.

**1a. Recompute the cid space (re-render-consistent).** cids derive from the sorted present-class set,
`start=2` (`reference_catalog`, `reference_seg_writer.py:99`). Read the dir's current
`cid_to_class` (`cid_to_class/cid_to_class_0000.pt`), apply the map to get new class names, then:

```python
# before: cid_to_class = {2:'amazon_green', 3:'amazon_pistachio', 4:'amazon_teal', 5:'amazon_tea', ...}
new_class   = {c: merge_map.get(cls, cls) for c, cls in cid_to_class.items()}   # old_cid -> new class
classes_new = sorted(set(new_class.values()))                                   # sorted present set
class_to_cid_new = {cls: i for i, cls in enumerate(classes_new, start=2)}       # re-render rule
lut = np.zeros(max(cid_to_class)+1, dtype=np.uint8)                             # old_cid -> new_cid
for old_cid, cls in new_class.items():
    lut[old_cid] = class_to_cid_new[cls]
# bg(0)/unlabeled(1) map to themselves: lut[0]=0, lut[1]=1
```

**1b. Remap every frame's `cid_mask`** (uint8, `cid_mask/cid_mask_NNNN.npy`) via the LUT â€” `iid_mask`
untouched:

```python
for f in range(n_frames):
    cid = ObsMask.deserialize_field(f, rd, "cid_mask").numpy()   # (H,W) uint8
    ObsMask(... cid_mask=tv_tensors.Mask(lut[cid]) ...).serialize(f, rd, only={"cid_mask"})
    # or write cid_mask/cid_mask_{f:04d}.npy directly under atomic_write_path()
```

**1c. Rewrite the once-per-dir catalogs** (per field, via `deserialize_field`â†’transformâ†’write). For a
merged class `N` from members `{A,B,...}`, pick **survivor = member name sorting first** and take
**all** of survivor's per-class entries; **concatenate** the list/row fields in a fixed order so
`class_to_name[N]` stays aligned with `class_to_l2w[N]` rows (the UFM adapter indexes
`class_to_l2w[c][n]` by `class_to_name` row, `optflow_isaac.py`):

| field (subdir) | transform |
|---|---|
| `cid_to_class` | rebuild from `class_to_cid_new` |
| `name_to_class` | re-value: `{name: merge_map.get(cls, cls)}` |
| `class_to_name` | merge: concat instance-name lists of merged members (fixed order) |
| `class_to_l2w` | merge: `torch.cat` instance rows in the **same** order as `class_to_name` |
| `class_to_reference`, `class_to_reference_depth`, `class_to_ref_intrinsics`, `class_to_ref_pose`, `class_to_ref` (RGBA) | keep **survivor's** entry for `N`; drop retired keys |
| `class_to_descriptors/<backbone>/` (both `DiftDescriptor` + `CleanDiftFinetunedDescriptor`) | keep survivor's per-class tensor for `N`; drop retired keys |
| `principal_components/<backbone>/` | refit per backbone via `fit_pca_basis(tokens, n=3)` over post-merge descriptors (viz only; cheap) |
| `obs_intrinsics`, `iid_to_name`, `iid_to_occlusion`, `cam2world`, `obs`, `observation_depth`, `iid_mask` | **unchanged** |

Idempotency: re-running with the same map is a no-op (present-class set already collapsed).

## Step 2 â€” Regenerate Stage-2 proposals + Stage-3 inliers (existing tools, no Isaac)

Proposals (`{class: coords}`) and inliers (`cid_mask == class_to_cid[cls]`) are class-keyed, so both
must be rebuilt under the new class names. Drop stale `verified_proposals/` + `verification/` (need a
retrained verifier), mirroring `stage-b-regen-reproj-gate.md`. Per dir (fast, ~7 frame/s, CPU):

```
RM=â€¦/reference_matching/src/reference_matching/configs ; ID=â€¦/isaac_datagen/src/isaac_datagen
RD=/data/user/jeffk/datasets/<ds>/render00N ; DS=$(dirname "$RD") ; IDX=<N>
EPS=<that dir's inlier_border_eps>     # mixed-persp 0.0, expanded-refseg 0.0, shelf-optflow 2.0
rm -rf "$RD"/{proposals,verified_proposals,verification}
isaac-datagen-proposals "$RD/runtime.yaml" dataset_dir="$DS" idx=$IDX \
  intrinsics_path=$ID/zed_K.npy proposer_config_path=$RM/grid_proposal.yaml \
  descriptor_config_path=$RM/descriptor.yaml proposer_device=cpu proposer_min_visible_ratio=0.3
isaac-datagen-inliers "$RD" --eps $EPS
```

(The reproj gate `gate_classes_reproj` is class-keyed via `name_to_class` + `class_to_l2w`, both
re-keyed in step 1, so it admits the merged class if any member instance is visible.)

## Verification

1. **Invariant â€” only labels changed:** assert `iid_mask`, `obs`, `observation_depth`, `cam2world`
   are byte-identical pre/post (sha256) on a sampled dir; only `cid_mask/` + the catalog subdirs differ.
2. **No cid/iid orphans:** `isaac-datagen-validate-obsmask` on each migrated dir (CidOrphan check,
   `validate_obsmask.py`) â€” every instance's cid must resolve.
3. **Coverage sanity:** spot-check inlier rate vs the pre-merge `stats/` (a near-dup merge should
   leave per-frame inlier counts ~unchanged, just re-bucketed under the merged class). Use the
   inlier viz (`inlier_figure`) on 2â€“3 frames per dataset.
4. **UFM round-trip:** instantiate `OptFlow2UFM` (`UFM-train/uniflowmatch/datasets/optflow_isaac.py`)
   on a migrated dir, pull a few samples for the merged class, and confirm no crash + the refâ†’obs
   warp (`class_to_l2w[c][n] @ class_to_ref_pose[c]`) projects the reference onto the instance
   (eyeball overlay) for instances sourced from **both** pre-merge members.
5. **Verifier smoke:** load one migrated dir through `RenderDirInlierFrameDataset` /
   `ImageInlierDataset` and confirm the merged class key serves without `KeyError`.

## Decision log / caveats

- **Residual migrate over re-render** â€” chosen because a near-dup merge changes no geometry; frames
  stay byte-identical, CleanDIFT descriptors are preserved (re-keyed), no Isaac/GPU, no
  finalize/black-frame footguns. Re-render remains the fallback if validation surfaces a catalog
  inconsistency the migrate can't reproduce.
- **Survivor selection must be consistent across ALL per-class fields** (reference + depth +
  intrinsics + pose + descriptors come from the same member) or the UFM warp/descriptor desync.
- **cid space is recomputed from the new sorted set**, so the migrated dir matches what a future
  re-render would bake (stays self-describing via `cid_to_class`).
- **`shelf-optflow` runtime.yaml has a stale absolute `objects_path`** (`/home/jeffk/repo/isaac_datagen/â€¦`,
  pre-workspace). It does not affect the migrate or Stage-2/3 (which read on-disk fields), but
  override it if any future re-render of those dirs is needed.
